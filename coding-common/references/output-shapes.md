# Output Shapes

## Bug / triage

- Symptom
- Repro status
- Root cause or strongest hypothesis
- Evidence
- Fix direction
- Verification gate
- Blockers / unknowns

## Implementation

- Changed
- Why
- Proof
- Residual risk

## Review gate

- Verdict: pass / pass with notes / changes requested / block
- Scope reviewed
- Findings, ordered by severity
- Missing evidence
- Proof checked
- Recommended next action

## Finding format

- Evidence: exact file/line/command/screenshot/log when available; use `path:line` for confirmed code findings
- Impact: why this matters in this repo
- Fix: concrete and minimal

## Repro pack

- Environment
- Steps
- Expected
- Actual
- Evidence links/screenshots/logs
- Verification criteria


## Evidence redaction

Redact secrets and unnecessary PII before including proof, logs, screenshots, request/response samples, or handoff artifacts.
