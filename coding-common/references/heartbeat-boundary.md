# Heartbeat Boundary

Heartbeat behavior is runtime policy, not ordinary coding workflow.

Coding skills must not start, resume, infer, or import tasks from a heartbeat alone.

Allowed on heartbeat only when already attached to active work:

- surface a real blocker
- surface a safety/security issue
- report a scheduled verification/check result
- stay quiet when there is no meaningful change

Do not use heartbeats as background project management, backlog scanning, or speculative coding triggers.
